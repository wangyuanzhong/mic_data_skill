# 开局确认包 + 指向性意向 / 数据能力 Gate

日期：2026-07-20  
状态：设计已口头确认各节；待用户审阅本文后进入 writing-plans  
范围：Skill / shared SOP 与包 README；**不改**分析与出图脚本逻辑

## 背景

当前 A（选标）与 C（指向性）各自步骤 0 收口，导致：

- 默认频段等未强制用户确认；
- 指向性相关确认偏晚；
- 「没 @ C 但口头要做 / @ 了 C 但无离轴」边界不清。

目标：开局一次确认默认值；用**意向 gate**与**数据能力 gate**协调是否跑 C；保持同批 A→B→C 串行。

## 决策摘要（已确认）

1. 开局确认包：公共项 +（按挂载）A/C 默认频段与 C 关注频点；**除「要不要指向性」外均带默认，必须用户确认**。
2. 未 @ C 时开局问「要不要做指向性？」；答「要」**视同已 @ C**。
3. 已 @ C：**不问**「做不做」；直接按要做执行。
4. 意向=要做，但探查无离轴：**软跳过 C**（提示一句），**不再问**是否强行做；继续 A/B（若有）。
5. 开局若要做 C：一并确认 C 默认（250–20000、`focus_freqs` 默认 `[1000]`）；探查后再只确认角度×文件名表。
6. 落地采用 **shared 开局确认包 + 各 Skill 专有 gate**（方案 1）。
7. 只挂 C、不挂 A：不问 A 的 100–15000 频段。
8. 脚本侧本设计不改；意向不进脚本必读 params。

## 1. 确认清单与时机

### 1.1 开局 Intake 确认包（探查前）

| 项 | 默认 | 何时出现 |
|----|------|----------|
| 分析根 | 用户路径 | 始终 |
| 产品名 | 可猜，须确认 | 始终 |
| 说明 | 问要不要加；不要→「无」 | 始终 |
| 幅度单位 | 可猜，须确认 | 始终 |
| 要不要做指向性？ | **无默认**（是/否） | **仅当未 @ C** |
| A 曲线频段 | 100–15000 Hz | 本批做选标时（已 @ A 或用户明确要做） |
| C 曲线频段 | 250–20000 Hz | 意向=要做 C（已 @ C，或未 @ 但答要） |
| C 关注频点 | `[1000]` | 同上；可加频点或改为 `[]` |

缺任一项已列必确认项 → 不探查、不建分析宽表。

### 1.2 探查后（仅意向=要做 C）

- 有离轴（≥2 个可区分角度，含轴向）→ 贴角度×文件名确认表（现有 1b）；确认前禁止写 C 宽表 / 跑 C 脚本。
- 无离轴 → 软提示跳过 C，不建或不再推进 C 产出；**不二次询问**。

### 1.3 明确不做

- 开局不确认具体角度列。
- 意向=不要 C 时：不问 C 频段 / focus_freqs，不跑 C。

## 2. Gate 与同批串行

### 2.1 意向怎么定

| 情况 | 意向 | 是否问「做不做指向性」 |
|------|------|------------------------|
| 已 @ C | 要做 | 不问 |
| 未 @ C，开局答要 | 要做（视同 @ C） | 问 |
| 未 @ C，开局答不要 | 不做 | 问 |

### 2.2 闸门顺序

1. 定意向（上表）。
2. Intake 确认包通过。
3. 探查 → 得 `has_off_axis`。
4. 若意向=要做且 `has_off_axis=false` → 软跳过 C。
5. 若意向=要做且 `has_off_axis=true` → 角度表确认 → 写 C params 角度字段 + 宽表 → C 脚本。

### 2.3 同批串行（保留并写清）

- 同时要 A（及 B）+ C：须 **A 全完**（含奇异值确认；要报告则 B 全完）后，才允许 **写 C 宽表 / 跑 C 脚本**。
- 允许开局先收意向与默认；**禁止**在等 A 确认的空档建 C 目录或写 C 表。
- A 与 C 产出目录物理隔离。
- 开局可先记意向；C 的 `params.json` 仅在真正开工 C（过数据能力 gate）时创建/写齐。

### 3.4 验收场景（手工）

1. 未 @ C，答不要 → 只 A（/B）。
2. 未 @ C，答要 → 视同 @ C；有离轴则串行后做 C。
3. 已 @ C，无离轴 → 软跳过，不问。
4. 已 @ C，有离轴，同批 A+B → A→B→C 串行。
5. 只挂 C → 不问 A 频段；确认 C 默认后探查→角度表或软跳过。
