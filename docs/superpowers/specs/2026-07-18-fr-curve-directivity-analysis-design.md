**日期:** 2026-07-18

| 项 | 名称 / 路径 |
|----|-------------|
| 新 Skill C（暂定） | `fr-curve-directivity-analysis` |

## 1. 目标与非目标

### 1.1 目标

一批测量数据中，除轴向频响外还有多角度频响（文件名区分）。Skill C 负责：

1. 与选标共用的 **intake + 探查确认 + 标准化**（经 `shared/`）  
2. 对每只样机、每个非轴向角度：计算 **角向 − 该机轴向** 的差值曲线  
3. 按 **同角度、多样机** 做差值一致性分析  
4. 用户提供包络框后，再做合格判定；无框则只做分析  
5. **本 Skill 内** 出正式报告（HTML→PDF），图/表/组版以脚本为主  

### 1.2 非目标

- **不依赖、不读取金标**（与 A 的金/辅标结果无关）  
- **不并进** A 的选标 SOP；不在 A 里嵌套整份 C  
- **正式报告不展示轴向曲线**（轴向仅用于计算，保留在 xlsx）  
- **不把 `shared/` 做成可触发的独立 Skill**（无独立 `description` 分流）

## 2. 包架构与协同机制

### 2.1 角色

| 层 | 角色 | 职责 |
|----|------|------|
| `shared/` | 共享库 | 公共 intake、探查/确认约定、标准化写出适配、公共 params 字段与 fill 模板 |
| A 选标 | 触发 Skill | 调 shared → 灵敏度 / 奇异值 / 金辅标；默认频段 100–15000（保持既有） |
| B 报告 | 触发 Skill | 仅服务 **选标** 产出的正式报告（不改其合同，除非另开任务） |
| C 指向性 | 触发 Skill | 调 shared → 差值 / 一致性 / 可选合格 → **本 Skill 内** HTML→PDF；默认频段 **250–20000** |

### 2.2 协同方式（平台无官方编排）

Agent Skills 规范只认「单目录一 Skill」；**没有** Skill 互调 API。本包协同靠：

1. **description 触发分流**  
2. **磁盘合同**（同级产出目录：`params.json` + `process.xlsx` + `process.md` 日志）  
3. **SOP 约定**：A/C 写明「标准化按 `shared/`」；禁止互相嵌套整份对方 SOP  

`shared/` 是同仓工程共享层。整包安装；不要只拷单个 Skill 却期望 shared 仍在。若未来单 Skill 分发，需拷贝 shared 或等待规范侧 `includes` 一类打包能力。

### 2.3 shared 放什么 / 不放什么

| 放进 `shared/` | 留在各自 Skill |
|----------------|----------------|
| 分析根、产品名、说明、幅度单位、产出目录约定 | **A：** 灵敏度中线、辅标数、奇异值/剔除、金辅标 |
| 探查 → 用户确认 → 标准化写出 | **C：** 轴向标签确认、包络/合格、差值与一致性、C 报告 |
| 公共 params 字段名与 fill 骨架前半 | 各 Skill 的默认 `f_lo_hz` / `f_hi_hz` |
| 角度×文件名确认表的 **版式**（`shared/references/`，见 §4） | |

### 2.4 兼容铁律（增量，不误伤 A）

1. **角度相关 params / sheet 只增量**；不改名、不复用、不破坏 A 既有 sheet 合同（如单表 `curves`、灵敏度/奇异值相关 sheet）。  
2. **A 继续消费既有「单表 `curves` 宽表」形态**，除非同一提交内把老脚本改到能吃新形态。  
3. **C 使用「一角度一 sheet」** 形态；与 A 的宽表形状可以不同——shared 允许 **两套标准化写出适配**。  
4. 若抽 shared 或改 sheet 数量/顺序/列位影响老脚本 → **必须同批改脚本并跑回归**（纯脚本问题，不靠改 Skill 文档假装兼容）。

## 3. Skill C — SOP 与闸门

**产出目录：** 与分析根同级 `<产品>_<YYYYMMDD_HHMMSS>/`（同 A；禁止仓库内 `output/`）。

| 步骤 | 内容 | 闸门 |
|------|------|------|
| 0 Intake | shared 收公共项 + C 追加命名/角度说明；建 `process.md` 骨架 + `params.json`；C 默认 `f_lo_hz=250`, `f_hi_hz=20000`（不主动问） | 缺必问项不进 1 |
| 1 探查映射 | 文件→样机序号→角度；标轴向参考角；写探查结论 | **确认前禁止宽表/脚本** |
| 1b 确认表 | 聊天展示固定格式表（§4）；不自然处 **必须主动提出** | **用户确认前禁止写出角度宽表、禁止进步骤 3** |
| 2 标准化 | 确认后 shared 写出 **一角度一 sheet**；同 sheet 内按 **序号** 排样机列；各角度 sheet 列位对齐 | 核对写入 md；未确认不写 xlsx |
| 3 计算 | 脚本：对列求差、一致性；有包络才合格 | 数字只来自脚本 sheet |
| 4 报告 | 脚本出图+组版；LLM 只产纯文本填空，作参数传入脚本 | HTML→PDF；报告不展示轴向曲线 |

闸门串：`0 → 1/1b 确认 → 2 → 3 → 4`。禁止跳步；禁止无 sheet 手补结论。

---

## 4. 角度×文件名确认表（格式写死）

- **列头 = 角度**（含轴向）  
- **行 = 序号**，单元格 = **文件名**  
- 版式模板放在 **`shared/references/`**（与现有 fill-\* 一致；非 `assets/`；A 的确认若无需角度列则用 shared 的简化变体或 A 自有 fill）  
- 示例：

| 序号 | axis | 90 | 180 |
|------|------|----|-----|
| 1 | …_0.csv | …_90.csv | …_180.csv |
| 2 | … | … | … |

Agent 确认时须主动提出例如：缺格、错位、同序号文件名对不上、某角度样机数与轴向不一致、疑似映射反了、序号重复/不连续。

**不要**单独的 `curve_map` 给脚本用。确认后列位不得偷偷重排。

---

## 5. 数据合同（C + shared 增量）

### 5.1 params

| 归属 | 字段 | 说明 |
|------|------|------|
| shared | `product`, `note`, `data_root`, `output_dir`, `unit`, `f_lo_hz`, `f_hi_hz`, `process_xlsx`, `process_md` | 公共 intake |
| shared（确认后） | `angles[]`（有序）、`axial_angle`、`sample_count` | 来自确认表 |
| A 专有 | `sensitivity.*`, `curves.aux_count`, `curves.exclude` | C 不写 |
| C 专有 | `envelope`（无则 `null`） | 无包络不算合格 |

### 5.2 process.xlsx — 标准化（C 路径）

| Sheet | 内容 |
|-------|------|
| 每角度一张（名=角度标签） | 频率 × 按序号排的样机列 |
| 可选 `angle_file_table` | 与确认表同构，便于复查；**非**求差所必需 |

脚本约定：各非轴向 sheet 与轴向 sheet **同序号同列位** → **对列求差**。

### 5.3 process.xlsx — C 计算（逐步留全）

| Sheet | 内容 |
|-------|------|
| `delta_<角度>` | 每非轴向角度一张：角向 − 轴向 |
| `consistency_<角度>`（或等价总表） | 同角度跨样机差值散度/包络宽度等 |
| `pass_<角度>` | **仅当** `envelope` 非空 |

轴向 sheet 保留在标准化层；报告不画轴向。

### 5.4 与 A 既有合同

A 的 `curves` 及分析 sheet **保持不变**。本设计新增的角度 sheet / C params **不得覆盖** A 合同。data-contract 文档在实现期增量更新（真源策略可随 shared 上移另定，不在本设计强制迁 B 的合同文件）。

---

## 6. 报告（C 内）

- 形态与 B 同款：**脚本出图 + HTML→PDF**。  
- **尽量脚本**：图、表、排版、PDF；**LLM 只生成纯文本填空**，写入参数（如 JSON 字段）再传给组版脚本；禁止 LLM 手拼整份 HTML/PDF。  
- 报告内容侧重：批次信息、确认表摘要、各角度差值、同角度一致性、（若有）合格结果。  
- **不展示轴向曲线。**  
- 模板 / outline 挂在 C 的 `references/`；可复用 B 的出图/组版**技术思路与依赖**，但不把选标报告 SOP 嵌进 C，也不要求用户先跑 B。

---

## 7. 错误处理与测试

### 7.1 错误

- 确认表未确认、同序号缺文件、各角度列数不一致 → 停在闸门。  
- 无包络 → 跳过合格 sheet 与报告合格结论。  
- 脚本失败 → 不手填分析数字。

### 7.2 测试

- C / shared：合成夹具覆盖对列求差、序号对齐、缺列失败、无包络跳过合格。  
- 抽 shared 或改写出路径后：**A 现有脚本测试必须回归**；破坏则同批修脚本。

---

## 8. 建议实现顺序（供 writing-plans）

1. 建 `shared/`：公共 intake + 探查/确认表 reference；A 步骤 0–2 改为调用 shared（**保持 A 宽表形态与老脚本兼容**）  
2. 建 C：一角度一 sheet 标准化适配 + 差值/一致性脚本 + 测试  
3. C 报告：文本参数 → 组版脚本 → HTML/PDF  
4. 更新包 README 边界表与流水线；增量 data-contract  
5. （可选后续）包络合格、真实批次验收

## 9. 已拍板摘要

| 议题 | 结论 |
|------|------|
| 与金标关系 | 无关 |
| 主计算 | 角向 − 同机轴向；同角度批一致性 |
| 合格 | 分析优先；用户给包络后再判 |
| 命名 | 无全球规则；单批强规律；探查 + 说明 + 确认 |
| 交付 | 完整逐步 xlsx + 详细 process.md + C 内正式报告 |
| 报告 | HTML→PDF；脚本为主；LLM 纯文本参数 |
| 频段默认（C） | 250–20000 Hz |
| 宽表 | 一角度一 sheet；按序号；确认表角度×文件名；脚本对列求差；无 curve_map |
| 架构 | 方案 1：shared 抬高；非独立标准化 Skill |
| 兼容 | 角度能力增量；动 sheet 必改老脚本并回归 |
