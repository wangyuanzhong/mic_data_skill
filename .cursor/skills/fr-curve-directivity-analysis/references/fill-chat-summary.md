# 步骤4：聊天短结（C 指向性）

差值与一致性 sheet 已由脚本写入、报告已出后，在聊天贴短结。

## 步骤0 intake：关注频点填法

默认 `focus_freqs=[1000]`。主动问用户："默认关注 1000Hz 差值，要不要加其他频点？"

- 用户给 `[1000, 4000]` → `params.focus_freqs` 写 `[1000, 4000]`
- 用户说"就 1000" → 写 `[1000]`
- 用户说"不关注" → 写 `[]`（合法跳过，报告无频点节）

## 必报项（缺一不可）

- 产出目录绝对路径
- 角度数（含轴向）、样机数
- 各非轴向角度的 `max_std_db` / `mean_std_db`（来自 `consistency_summary`）
- 报告 `report.pdf` 绝对路径
- `figures/` 中差值叠图张数
- 自检是否全部通过

## 禁止

- 贴整张 sheet 表
- 手写报告数字
- 假装合格（`envelope=null` 时不得说「合格/不合格」）
