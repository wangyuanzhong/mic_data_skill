# 角度 × 文件名确认表（聊天展示；格式写死）

用于指向性 Skill 步骤 1b：探查后、写角度宽表前，**必须在聊天里贴这张表请用户确认**。

## 格式（写死）

- **列头 = 角度**（含轴向）
- **行 = 序号**，单元格 = **文件名**
- 序号从 1 起，连续

示例：

| 序号 | axis | 90 | 180 |
|------|------|----|-----|
| 1 | …_0.csv | …_90.csv | …_180.csv |
| 2 | … | … | … |

## 确认时必须主动提出的不自然处

Agent 在请用户确认时，若发现下列任一情况，**必须主动指出**，不得默默按表继续：

- 缺格（某序号在某角度列无文件）
- 错位（同序号不同角度的文件名对不上同一只样机）
- 某角度样机数与轴向不一致
- 疑似映射反了（如 90 与 180 列互换）
- 序号重复或不连续
- 文件名里角度标签与列头矛盾

## 硬规则

- **用户确认前：禁止写出角度宽表，禁止进步骤 3 计算。**
- 确认后列位不得偷偷重排；若要改序号或角度顺序，须重新贴表确认。
- **不要**单独的 `curve_map` 给脚本用；脚本直接对宽表各角度 sheet 同列位求差。
